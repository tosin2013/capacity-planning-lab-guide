= Strategic Capacity Planning & Forecasting for OpenShift at Scale

A full-day, highly interactive workshop designed to help platform engineering and development teams transition from reactive "firefighting" to predictive, data-driven capacity management using OpenShift and Red Hat Advanced Cluster Management (RHACM).

== Workshop Overview

This workshop teaches strategic capacity planning through 7 comprehensive modules covering:

* **Module 1**: The Planning Horizon & 90-Day Audit - Establish capacity baselines
* **Module 2**: Mathematics of Forecasting - Pod Velocity Model and PromQL
* **Module 3**: Developer Track - QoS classes, right-sizing, and HPA
* **Module 4**: Infrastructure Track - Node density, etcd limits, split vs grow decisions
* **Module 5**: Fleet Observability (RHACM) - Multi-cluster dashboards and metric allowlists
* **Module 6**: Integration Challenge - Black Friday chaos game simulation
* **Module 7**: Strategic Roadmapping - 12-month capacity planning and executive communication

**Duration**: 8 hours (full-day workshop)

**Target Audience**: Platform Engineers, SREs, Application Developers, Architects, FinOps practitioners

== Accessing the Workshop

== Local Development

=== Using Containers (Recommended)

Test and develop the lab guide locally using Podman or Docker:

[source,sh]
----
podman run --rm --name antora -v $PWD:/antora -p 8080:8080 -i -t ghcr.io/juliaaano/antora-viewer
----

For SELinux environments:

[source,sh]
----
podman run --rm --name antora -v $PWD:/antora:z -p 8080:8080 -i -t ghcr.io/juliaaano/antora-viewer
----

Then open http://localhost:8080 in your browser.

=== Using Utilities (Alternative)

[source,sh]
----
./utilities/lab-serve
----

Open http://localhost:8080 in your browser.

To rebuild HTML:
[source,sh]
----
./utilities/lab-build
----

== Editing Workshop Content

Workshop content is located in `content/modules/ROOT/pages/`:

[source,sh]
----
./content/modules/ROOT/pages/
├── index.adoc           # Workshop introduction and overview
├── module-01.adoc       # Module 1: The Planning Horizon
├── module-02.adoc       # Module 2: Mathematics of Forecasting
├── module-03.adoc       # Module 3: Developer Track
├── module-04.adoc       # Module 4: Infrastructure Track
├── module-05.adoc       # Module 5: Fleet Observability
├── module-06.adoc       # Module 6: Integration Challenge
└── module-07.adoc       # Module 7: Strategic Roadmapping
----

Many modern editors such as Visual Studio Code offer live AsciiDoc preview extensions.

== Workshop Attributes

The workshop uses Antora attributes for dynamic content substitution. When deploying via RHDP, these are automatically populated:

* `{guid}` - Student's unique workshop instance identifier
* `{cluster_domain}` - OpenShift cluster ingress domain
* `{openshift_console}` - OpenShift web console URL
* `{namespace}` - Student's workshop namespace
* `{rhacm_console}` - RHACM console URL
* `{grafana_url}` - Observability Grafana URL

For local development or GitHub Pages, these use placeholder values.

Deploy just the Showroom lab guide to an existing OpenShift cluster:

[source,yaml]
----

kind: Deployment

  name: showroom
spec:
  template:
    spec:
      initContainers:
      - name: git-cloner
        image: quay.io/rhpds/git-cloner:v1.1.2
        env:
        - name: GIT_REPO_URL
          value: https://github.com/tosin2013/capacity-planning-lab-guide.git
        - name: GIT_REPO_REF
          value: main
      # ... (see AgnosticD workload for complete configuration)
----
